# Architecture Decision Records (ADR)

Đây là nơi ghi lại các quyết định kiến trúc, công nghệ và design quan trọng của dự án từ **góc nhìn cá nhân**.

## Mục đích

- Giúp AI hiểu **tại sao** chúng ta chọn giải pháp hiện tại.
- Tránh lặp lại các quyết định sai trong tương lai.
- Dễ onboarding và review code.

## Quy tắc đặt tên file

- Số thứ tự: `001-`, `002-`, …
- Tên file mô tả ngắn gọn (kebab-case).
- Chỉ ghi quyết định **quan trọng** (không ghi mọi thứ nhỏ).

**Khi tạo ADR mới:** copy từ `template.md`.

# Execution Logs

Use this folder for traceable run logs per task.

## Naming

- `YYYYMMDD-<task-id>-worklog.md`
- Example: `20260414-sim-demo-1-worklog.md`

## Required content

1. Context and objective
2. Step-by-step actions
3. Commands and outputs summary
4. Validation evidence (tests/checks)
5. Issues, fixes, and remaining risks

## Rule

- No secrets or credentials in logs.
- Keep logs concise but auditable.

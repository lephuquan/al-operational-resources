# Task execution logs

This folder stores detailed execution logs for completed or in-progress tasks.

## Naming

- `YYYYMMDD-<scope>-worklog.md`
- Example: `20260414-sim-demo-1-2-worklog.md`

## Recommended sections

1. Context and objective
2. Step-by-step actions (commands + file changes)
3. Validation and evidence
4. Issues and fixes
5. Lessons learned / improvements for next tasks

Keep logs concise but traceable; avoid secrets.

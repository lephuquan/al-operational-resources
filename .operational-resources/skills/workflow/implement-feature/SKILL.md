# Skill: Implement Feature (End-to-End)

## Goal

Hoàn thành một tính năng có kiểm soát: spec → thiết kế → code → test → docs.

## Steps

1. Đọc `docs/current-task/<task>.md` và `docs/specs/feature-*.md`.
2. Thiết kế ngắn (API + DB + error codes) — ghi trong task nếu cần.
3. Triển khai theo lớp: DTO → service → repository → controller.
4. Test: unit + integration tối thiểu.
5. Cập nhật `docs/api/*` và `rules/` nếu đổi contract.
6. Self-review theo `code-quality/review-code/checklist.md`.

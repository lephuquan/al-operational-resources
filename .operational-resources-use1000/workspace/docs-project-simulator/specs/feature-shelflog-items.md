# Feature: ShelfLog — shelf items (FEA-SHELF-001)

**Feature code:** FEA-SHELF-001  
**Status:** Active (demo reference application)  
**Canonical brief:** `.operational-resources/simulator/DEMO-PROJECT-BRIEF.md`  
**Last updated:** 2026-04-19

## 1. Summary

Expose versioned REST APIs under **`/api/v1/shelf-items`** to create, read, update, delete, and list **ShelfItem** records. The demo targets **PostgreSQL** when running the app locally (Docker Compose) and **H2** for automated tests. **No authentication** in v1 — local/demo only.

## 2. Personas & goals

| Persona | Goal |
|---------|------|
| Developer / power user | Maintain a simple inventory of items on a personal “shelf” via HTTP |
| AI-assisted implementer | Implement CRUD + pagination + validation with clear AC and tests |

## 3. Domain model

### 3.1 Entity: ShelfItem

| Field | Type | Rules |
|-------|------|--------|
| `id` | UUID | Generated by server on create |
| `title` | string | Required; length **1–200** after trim |
| `category` | enum | Required; one of `OFFICE`, `HOBBY`, `KITCHEN`, `OTHER` |
| `quantity` | integer | Required; range **0–999** inclusive |
| `notes` | string | Optional; max **2000** characters |
| `createdAt` | `Instant` | Set on create; immutable |
| `updatedAt` | `Instant` | Set on create and on each successful update |

### 3.2 Business rules

- **BR1:** `quantity` must not exceed **999** (API must return **400** when violated).
- **BR2:** `title` must not be blank after trimming whitespace.
- **BR3:** **Hard delete** — `DELETE` removes the row; subsequent `GET` returns **404**.

## 4. API behavior (v1)

**Base path:** `/api/v1/shelf-items`

| Method | Path | Behavior |
|--------|------|----------|
| POST | `/api/v1/shelf-items` | Create; **201** + response body including `id` and timestamps |
| GET | `/api/v1/shelf-items/{id}` | **200** if found; **404** if not |
| PUT | `/api/v1/shelf-items/{id}` | Full replace of mutable fields (`title`, `category`, `quantity`, `notes`); **404** if not found; validation as on create |
| DELETE | `/api/v1/shelf-items/{id}` | **204** empty body; **404** if not found |
| GET | `/api/v1/shelf-items` | Paginated list; query `page` (0-based), `size` (default **20**, max **100**); optional filter `category` (exact enum match) |
| GET | `/api/v1/shelf-items/export` | CSV export; same `page`, `size`, `category` as list; UTF-8 comma-separated, no BOM; attachment filename `shelf-items-<UTC date>.csv`. If **total matching rows** (same filter, no pagination cap) **exceeds 5000**, respond **413** with JSON error code **`SHELF_413_EXPORT`** and **no** CSV body. Text fields that start with `=`, `+`, `-`, or `@` are prefixed with `'` in the CSV cell to reduce spreadsheet formula injection risk. |

### 4.1 Request body (create / update)

JSON fields: `title`, `category`, `quantity`, `notes` (omit or `null` for `notes` per implementation — document chosen convention in API layer and stay consistent).

### 4.2 List response shape

Use a **stable** paginated JSON shape, for example:

```json
{
  "content": [ /* ShelfItemResponse */ ],
  "page": 0,
  "size": 20,
  "totalElements": 42
}
```

Spring Data `Page` serialization is acceptable if globally consistent; otherwise use an explicit DTO. The same shape must be reflected in `docs/api/08-endpoint-list.md` and tests.

### 4.3 Errors

- **400** — validation failures; prefer field-level detail (Bean Validation and/or shared error envelope — see `docs/api/05-error-handling.md`).
- **404** — resource not found by `id`.
- **413** — CSV export refused when the filtered catalogue has **more than 5000** matching rows (`SHELF_413_EXPORT`); client should narrow filters (for example `category`) and retry.

### 4.4 CSV export columns (stable order)

Header row (first line): `id`, `title`, `category`, `quantity`, `notes`, `createdAt`, `updatedAt` (ISO-8601 instants). One data row per shelf item in the current page, sorted like the list API (`createdAt` descending).

**409** is not required for v1.

## 5. Non-functional

| Area | Requirement |
|------|----------------|

| Performance | List endpoint must use **database pagination** (no full-table load) |
| Observability | Expose **`GET /actuator/health`** for demo operations |
| Dev DB | PostgreSQL via `.operational-resources/simulator/docker-compose.postgres.yml` |
| Test DB | H2, profile `test` |

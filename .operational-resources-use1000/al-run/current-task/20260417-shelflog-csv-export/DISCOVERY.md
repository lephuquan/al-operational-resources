## 2) Problem framing (must be concrete)

- Business goal: Allow operators to download a **CSV** of shelf items for offline review or handoff to spreadsheets, without building a full reporting product.
- In-scope: One **GET** (or **POST** if body filter required) endpoint under `/api/v1/` that returns a CSV file for the **ShelfLog demo** dataset; UTF-8; respects existing pagination/filter semantics where practical.
- Out-of-scope: Scheduled jobs, email delivery, S3 upload, Excel binary format, auth beyond “demo public”, changing ShelfLog CRUD semantics.
- Success criteria: Human can call the endpoint in local demo and open the file in a spreadsheet tool; documented limits (max rows / timeout) in task + API changelog.
